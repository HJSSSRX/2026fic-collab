## 分析策略

### 第一步：解压手机备份
```bash
mkdir extracted
tar -xf "检材2-手机.tar" -C extracted/
```
然后浏览目录结构，确定是什么格式的备份（Android备份/物理提取/逻辑提取）。

### 推荐分析顺序

**Phase 1 — 快速信息收集（Q1-Q10）**
1. **设备信息** → 手机型号（Q1）— 查找 `build.prop` 或设备信息文件
2. **日历/行程** → 迪拜出行日期（Q2）— 日历数据库或旅行APP
3. **聊天APP** → 安装日期、数据库名、解密密码（Q3/Q4/Q5）— 可能是 Telegram/Signal/WhatsApp
4. **加密聊天记录** → 云服务器商家钱包、转账hash（Q6/Q7）
5. **AI软件** → 提问次数、模型版本（Q8/Q9）
6. **无人机数据** → DJI飞行日志 → GPS坐标 → 县名（Q10）

**Phase 2 — APK逆向（Q11-Q17）**
1. 找到可疑视频类APP的APK文件
2. `jadx` 反编译，分析 `AndroidManifest.xml`（Q11权限）
3. 分析启动Activity → 加载的URL和离线页面（Q12）
4. 搜索网络请求代码 → 上传地址编码方式（Q13）
5. 搜索SQLite相关代码 → 表名（Q14）
6. 解密 `assets/config.dat` → USDT钱包（Q15）
7. 搜索 `@JavascriptInterface` → JS Bridge方法（Q16）
8. 搜索备用服务器逻辑 → 域名和端口（Q17）

### 关键搜索位置
- `/data/data/` — APP私有数据
- `/sdcard/` — 用户文件（照片、下载）
- `/data/app/` — 已安装APK
- DJI日志: `/sdcard/DJI/` 或 `/data/data/dji.*/`
- AI软件: 搜索 `ollama`, `chatgpt`, `deepseek` 等关键词
